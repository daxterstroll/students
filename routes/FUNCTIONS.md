# Довідник по всіх функціях проєкту

Автоматично зібрано з докстрінгів кожної функції (після оновлення). Маршрути Flask показані поруч із назвою функції.

## admin.py — адмін-розділи

- **`manage_study_periods_bulk_assign()`** `/admin/study_periods/bulk_assign` -> 
  Масове призначення періоду навчання (філія, дати, група) одразу декільком студентам — обраній групі цілком та/або довільному списку студентів.

- **`manage_diplomas()`** `/admin/manage_diplomas` -> 
  Перегляд і масове збереження номерів диплома/додатка для всіх студентів обраної групи.

- **`manage_accreditations()`** `/admin/manage_accreditations` -> 
  CRUD-сторінка довідників акредитацій (ступінь + спеціальність + текст українською/англійською) для підстановки в документи.

- **`manage_education_documents()`** `/admin/manage_education_documents` -> 
  CRUD-сторінка документів про освіту студентів (атестат/диплом попереднього рівня) разом із додатковими даними про визнання іноземного документа (foreign_education_docs).

- **`manage_groups()`** `/admin/manage_groups` -> 
  CRUD-сторінка навчальних груп: додавання, редагування та видалення (видалення заборонено, якщо є пов'язані студенти/предмети/активності).

- **`manage_subjects()`** `/admin/manage_subjects` -> 
  CRUD-сторінка навчальних предметів групи: додавання/редагування/видалення/переміщення по позиції, а також масове виставлення оцінок студентам з цього предмету.

- **`manage_activities()`** `/admin/manage_activities` -> 
  CRUD-сторінка практик/курсових/атестацій (спільна логіка для трьох типів activity-сутностей через ALLOWED_TABLES) з масовим виставленням оцінок.

- **`view_logs()`** `/admin/view_logs` -> 
  Показує журнал дій (app.log) у зручному розібраному вигляді: дата/час/рівень/користувач/дія, з можливістю фільтрації на фронтенді.

- **`manage_users()`** `/admin/users` -> 
  Список користувачів системи та редагування їхніх прав доступу (is_admin + список дозволів permissions).

- **`add_user()`** `/admin/users/add` -> 
  Форма створення нового користувача (логін/пароль/роль/групи).

- **`edit_user(user_id)`** `/admin/users/<int:user_id>/edit` -> 
  Форма редагування ролі та прив'язаних груп існуючого користувача.

- **`change_password(user_id)`** `/admin/users/<int:user_id>/change-password` -> 
  Форма зміни пароля вказаного користувача адміністратором.

- **`delete_user(user_id)`** `/admin/users/<int:user_id>/delete` -> 
  Видалення користувача та його зв'язків з групами (user_groups).

- **`group_export()`** `/admin/group_export` -> 
  Сторінка вибору параметрів (група і/або рік народження, шаблон .docx) перед масовою генерацією документів студентів.

- **`allowed_file(filename)`** 
  Перевіряє, чи має файл дозволене розширення (.xlsx) для імпорту.

- **`import_subjects()`** `/admin/import_subjects` -> 
  Імпорт списку предметів групи з Excel-файлу.

- **`generate_group_docs()`** `/admin/generate_group_docs` -> 
  Генерує .docx-документи (за обраним шаблоном) для всіх студентів, що підпадають під фільтр (група і/або рік народження), і повертає їх одним ZIP-архівом.

- **`archive_group(group_id)`** `/admin/archive/<int:group_id>` -> 
  Архівує групу разом з усіма її студентами (archived=TRUE).

- **`unarchive_group(group_id)`** `/admin/unarchive_group/<int:group_id>` -> 
  Повертає архівну групу разом зі студентами назад в активні (archived=FALSE).

- **`archive()`** `/admin/archive` -> 
  Показує список заархівованих груп та їхніх студентів.

- **`save_preview_to_file(preview)`** 
  Зберігає проміжний результат парсингу Excel-файлу імпорту документів про освіту у тимчасовий JSON-файл (щоб не тримати великі дані в сесії) і повертає його ID.

- **`load_preview_from_file(preview_id)`** 
  Завантажує раніше збережений (save_preview_to_file) прев'ю-результат імпорту за його ID.

- **`fuzzy_find_student(cursor, full_name, threshold)`** 
  Нечіткий пошук студента за повним ім'ям (ПІБ) серед активних студентів, використовуючи rapidfuzz; повертає ID найкращого збігу, якщо він проходить поріг схожості threshold.

- **`translate_to_en(text)`** 
  Перекладає текст на англійську через GoogleTranslator з кешуванням результатів (translation_cache), щоб не робити повторні запити до перекладача для однакового тексту.

- **`find_country(text)`** 
  Визначає країну (укр./англ. назву) за ключовими словами в тексті установи/довідки; за замовчуванням вважає, що це Україна.

- **`parse_document(text)`** 
  Розбирає текстовий опис документа про освіту (формат 'Тип документа Номер; ДД.ММ.РРРР; Ким видано: Установа') на структуровані поля (тип, номер, дата, установа, країна) для імпорту.

- **`parse_reference_cell_ua(text)`** 
  Розбирає комірку Excel з даними довідки про визнання документа (номер; установа; країна; дата) на окремі поля.

- **`parse_recognition_cell_ua(text)`** 
  Розбирає комірку Excel з даними про визнання (номер сертифіката; орган; дата) на окремі поля.

- **`import_documents_preview(file_path, db)`** 
  Читає Excel-файл з документами про освіту, для кожного рядка знаходить студента (fuzzy_find_student), парсить документ (parse_document) та формує прев'ю-список змін (нові/оновлення) для підтвердження користувачем перед комітом у import_docs_commit.

- **`import_docs_preview()`** `/admin/import_education_docs_preview` -> 
  Приймає завантажений Excel-файл, будує прев'ю через import_documents_preview і показує сторінку підтвердження перед фактичним імпортом.

- **`import_docs_commit()`** `/admin/import_docs_commit` -> 
  Записує в БД документи про освіту (та дані про визнання), підтверджені користувачем на сторінці прев'ю (тільки позначені чекбоксом рядки).


## students.py — робота зі студентами

- **`student_list()`** `/students` -> 
  Головна сторінка списку студентів: пошук, фільтр по групі, пагінація, сортування (в т.ч. українська колація для ПІБ), а також обмеження видимості для не-адмінів лише їхніми групами. Для кожного студента одразу рахує заповненість особистих даних, військового обліку, оцінок та активностей (для індикаторів прогресу в UI).

- **`student_details(student_id)`** `/students/<int:student_id>` -> 
  Картка студента: особисті дані, військовий облік, оцінки з предметів/практик/курсових/атестацій, документи про освіту та періоди навчання.

- **`manage_study_periods(student_id)`** `/students/<int:student_id>/study_periods` -> 
  CRUD-сторінка періодів навчання (філія, дати, примітка) конкретного студента.

- **`add_student()`** `/students/add` -> 
  Форма додавання нового студента (особисті дані + прив'язка до групи).

- **`edit_student(student_id)`** `/students/<int:student_id>/edit` -> 
  Форма редагування особистих даних студента.

- **`delete_student(student_id)`** `/students/<int:student_id>/delete` -> 
  Видаляє студента разом з усіма пов'язаними записами (military, grades, activity_grades, education_documents, diplomas).

- **`add_military(student_id)`** `/students/<int:student_id>/military/add` -> 
  Форма додавання даних військового обліку студенту, який ще їх не має.

- **`military_data(student_id)`** `/students/<int:student_id>/military` -> 
  Форма перегляду/редагування наявних даних військового обліку студента.

- **`delete_military(student_id)`** `/students/<int:student_id>/military/delete` -> 
  Видаляє запис військового обліку студента.

- **`generate(student_id)`** `/students/<int:student_id>/generate` -> 
  Генерує .docx-документ для одного студента за обраним шаблоном та завантажує його користувачу.

- **`edit_activities_grades(student_id)`** `/activities_grades/<int:student_id>` -> 
  Масове виставлення оцінок студентам групи з практик/курсових/атестацій (аналог admin.manage_activities, але з боку картки студента/групи).

- **`edit_grades(student_id)`** `/grades/<int:student_id>` -> 
  Форма виставлення/редагування оцінок одного студента з усіх предметів його групи.

- **`import_from_excel()`** `/import_from_excel` -> 
  Імпорт студентів з Excel-файлу.


## auth.py — логін / логаут

- **`index()`**
  Коренева сторінка сайту - просто перенаправляє на форму логіну.

- **`login()`**
  Форма входу в систему. GET показує форму (або редіректить залогіненого користувача), POST перевіряє логін/пароль, заповнює сесію і логує подію "ввійшов у систему"/"невдала спроба входу".

- **`logout()`**
  Завершує сесію користувача, логує подію "вийшов із системи" та повертає на форму логіну.

## utils.py — спільні утиліти, логування, декоратори прав

- **`log_action(username, action, group_ids, mode, details)`**
  Записує дію користувача в лог-файл (app.log) у вигляді:
"YYYY-MM-DD | HH:MM:SS | INFO | 👤 username - дія (групи: ...) | деталі"

Параметри:
    username   - ім'я користувача, що виконав дію (див. helpers.current_username())
    action     - короткий опис дії, наприклад "додав студента"
    group_ids  - ID груп, яких стосується дія (для не-адмінів ці ID
                 перетворюються на людські назви груп)
    mode       - необов'язковий режим/контекст дії
    details    - додаткові деталі (наприклад, кількість оброблених рядків)

ВАЖЛИВО: раніше ця функція не мала обробки помилок. Якщо get_db()
не міг відкрити файл БД (немає прав доступу, диск переповнений тощо)
або запит до groups падав, виняток "вилітав" з log_action() і міг
зламати весь HTTP-запит - навіть якщо основна дія (наприклад,
збереження студента) вже завершилась успішно. Тепер збій самого
логування ніколи не ламає основний функціонал, але обов'язково
фіксується як ERROR в лог, щоб про проблему було відомо.

- **`login_required(role)`**
  Декоратор: пускає на сторінку тільки авторизованих користувачів.
Стара версія для зворотної сумісності - для нових перевірок прав
краще використовувати permission_required() нижче.

Args:
    role (str, optional): якщо вказано (напр. 'admin'), доступ
        дозволено лише користувачам з саме такою роллю; інакше -
        403 Forbidden. Якщо None/'' - достатньо просто бути залогіненим.

Поведінка при відсутності сесії: редірект на сторінку логіну.

- **`permission_required(permission)`**
  Декоратор: перевіряє авторизацію та (за потреби) конкретний дозвіл
користувача (permission) із таблиці users.permissions (JSON-список).

- Якщо permission=None: достатньо просто бути залогіненим.
- Якщо permission вказано: потрібен is_admin=1 АБО цей дозвіл
  присутній у списку permissions користувача.
- Значення is_admin/permissions спочатку читаються з сесії (кеш),
  і лише якщо їх там ще немає - підвантажуються з БД один раз і
  кешуються в сесії (щоб не робити зайвий SELECT на кожен запит).

Незалогінених користувачів редіректить на логін;
користувачів без потрібного дозволу - на список студентів.

- **`transliterate_ukrainian(text)`**
  Транслітерація українського тексту згідно з Постановою КМУ №55-2010
https://zakon.rada.gov.ua/laws/show/55-2010-п

Використовується для автоматичного заповнення англомовних версій
прізвища/імені студента при імпорті з Excel (див. generate_english_name).
Повертає порожній рядок, якщо на вхід передано не-рядок або None.

- **`generate_english_name(last_name_ua, first_name_ua)`**
  Транслітерує українські прізвище та ім'я в англійський варіант
(використовуючи transliterate_ukrainian). Викликається при імпорті
студентів з Excel, коли англомовних ПІБ немає у файлі.

Повертає кортеж (last_name_eng, first_name_eng).

- **`get_available_templates()`**
  Повертає список шляхів (у форматі 'template_word/файл.docx') до всіх
.docx-шаблонів у папці template_word - вони показуються користувачу
як варіанти шаблону при масовій генерації документів
(admin.group_export / admin.generate_group_docs).

Якщо папки template_word не існує - повертає порожній список
(а не кидає помилку).


## helpers.py — нові спільні хелпери (усувають дублювання коду)

- **`current_username()`**
  Повертає ім'я поточного користувача з сесії для логів/повідомлень. Замінює 48 повторів `session.get('username', 'невідомо')`.

- **`sort_ukrainian(items, key_func)`**
  Безпечно сортує список за українською локаллю з фолбеком на звичайний порядок, якщо локаль не встановлена на сервері (замість падіння).

## db.py — підключення до БД та колація

- **`ukrainian_collation(str1, str2)`**
  Функція колації для SQLite (`COLLATE UKRAINIAN` в SQL). Локаль тепер встановлюється один раз при старті модуля, а не на кожне порівняння.

- **`get_db()`**
  Створює нове з'єднання sqlite3 до students.db з row_factory=Row, колацією UKRAINIAN та увімкненими foreign keys.

## init_db.py

Одноразовий скрипт створення бази даних "з нуля" з усіма таблицями та користувачем admin/admin123. Не містить окремих функцій (виконується як послідовність команд при запуску `python init_db.py`).

## update_groups.py — щоденний cron-скрипт

- **`update_groups()`**
  Раз на рік, 1 вересня, переводить активні групи на наступний курс або архівує їх, якщо навчання завершено. В інші дні нічого не робить.

## gen_docx.py — генерація .docx документів студента

- **`_to_richtext_multiline(text, separator, font_size_pt, font_name)`**
  Превращает 'Назва 1; Назва 2' в RichText с реальным переносом строки
после каждого separator, с заданным размером и шрифтом.

- **`_format_date_ddmmyyyy(date_str)`**
  Приводит дату из БД (YYYY-MM-DD, DD.MM.YYYY или уже DD/MM/YYYY) к формату DD/MM/YYYY.

- **`get_study_periods(cursor, student_id, lang, default_filiya, default_filiya_en)`**
  Единый хелпер для подстановки в docx-шаблон.
Возвращает кортеж (names_text, dates_text):
  names_text -> "Назва 1; Назва 2"
  dates_text -> "01/09/2022-31/05/2025; 01/09/2025-31/05/2026"

lang='ua' -> використовує filiya
lang='en' -> використовує filiya_en

Якщо записів немає — повертає дефолтну філію і порожні дати.

- **`insert_subjects_table(doc, student_id)`**
  Вставка таблицы с предметами и оценками студента.

- **`clean_text(text)`**
  Очищает текст от непечатаемых символов, сохраняя переносы строк и тире (—), и приводит к строке.

- **`format_grade(grade, subject_type)`**
  Преобразует числовую оценку в текстовую форму по формуле.

- **`get_subjects_grades(student_id, group_id)`**
  Получение данных о предметах и их оценках.

- **`get_practice_data(student_id, group_id)`**
  Получение данных о практиках и их оценках.

- **`get_coursework_data(student_id, group_id)`**
  Получение данных о курсовых работах и их оценках.

- **`get_attestation_data(student_id, group_id)`**
  Получение данных об аттестациях и их оценках.

- **`gen_doc(student, military, template, out, user_name)`**
  Генерирует документ для студента на основе шаблона.
